Dice Loss — Analysis Notes


1. Objective
Dice Loss is a region-based loss function designed to maximize overlap between predicted segmentation and ground truth.

LDice=1−((2⋅∣P∩G∣)/(∣P∣+∣G∣))
	​

Where:
P=predicted region
G=ground truth region


2. Motivation in This Task
Previous losses (CE, WCE, Focal, Weighted Focal) operate at the pixel level and fail under extreme class imbalance.

Dice Loss was introduced to:
Directly optimize region overlap and reduce dependence on pixel frequency.


3. Observed Performance
Stable training across epochs
Comparable global performance to Focal and Weighted Focal

Observed:
IoU ≈ 0.43–0.44  
F1 ≈ 0.52–0.54


4. Per-Class Behavior
Strong performance on dominant classes (class 7, class 14)
Significant improvement on moderately frequent class (class 21 ≈ 0.85)
No improvement on extremely rare classes (still ≈ 0 accuracy)


5. Key Observation
Dice Loss improves moderately imbalanced classes but fails on extremely rare classes.


6. Interpretation
Dice Loss operates on region overlap, not pixel-wise classification.

This results in:
Normalization by region size rather than pixel count.

Thus:
Classes with sufficient spatial presence contribute meaningfully
Classes with extremely low representation contribute negligibly


7. Why Dice Performs Better than CE/Focal
Removes dominance of pixel frequency
Treats predictions at a region level
Reduces bias toward dominant classes

This explains recovery of class 21 (similar to WCE behavior).

8. Limitation
Dice Loss still depends on
Existence of meaningful regions for each class.

For extremely rare classes:
Insufficient pixels to form stable regions
Overlap computation becomes ineffective

Thus:
Rare classes fail to generate meaningful gradient signals.


9. Comparison with Previous Losses
Loss	        Rare Classes	Moderate Classes	Global Metrics
CE	            ❌	          ❌	                 Moderate
WCE	            ❌	          ✔	                  Moderate
Focal	        ❌	          ❌	                 High
Weighted Focal	❌ (worse)	  ❌	                 Highest
Dice	        ❌	          ✔ (stable)	      High


10. Critical Insight
Region-based normalization reduces class imbalance effects but does not eliminate them under extreme imbalance.


11. Conclusion
Dice Loss improves learning for moderately frequent classes
Provides stable and strong global performance
Fails to address extremely rare classes
Dice Loss is more effective than classification-based losses but remains insufficient for extreme pixel-level imbalance.


12. Motivation for Next Approach
The limitations of Dice Loss indicate the need for:
Explicit class-volume weighting
Control over false positives vs false negatives
Stronger handling of extreme imbalance
This motivates the use of Generalized Dice Loss and Tversky-based losses.


Key takeaway line (VERY IMPORTANT)
Dice Loss mitigates class imbalance at the region level, but fails when class presence is too sparse to form meaningful spatial structures.
